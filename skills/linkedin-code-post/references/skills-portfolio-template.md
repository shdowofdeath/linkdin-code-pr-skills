# Skills Portfolio Post Template

## Template Structure

```
{hook_line}

{top_skills_with_context}

{currently_learning}

{cta_question}

{hashtags}
```

## Field Guidelines

### hook_line
Open with a reflection on your journey or a framing that makes a skills post feel personal rather than like a resume dump.

Good examples:
- "I realized I've never actually mapped out what I know. So I did. Here's what I found."
- "After 4 years of building, here's what actually stuck in my toolkit."
- "Someone asked me 'what's your stack?' last week. The honest answer was longer than I expected."
- "I spent this weekend auditing my own skills. It was equal parts humbling and encouraging."

Bad examples:
- "Here are my technical skills:" (resume format)
- "I know a lot of technologies" (says nothing)
- "Skills update!" (not engaging)

### top_skills_with_context
6-10 lines showcasing your strongest skills WITH context. Don't just list technologies -- show evidence and experience.

Format each skill with context:
- "**TypeScript** -- 3 years across 5 projects, from React frontends to Node.js APIs"
- "**Python** -- data pipelines, FastAPI backends, and automation scripts"
- "**PostgreSQL** -- my default database. Comfortable with query optimization, migrations, and replication"
- "**Docker + Kubernetes** -- containerized everything I've built in the last 2 years"

Group related skills together:
- "Frontend: React + TypeScript. I've tried Vue and Svelte too, but React is where I'm most productive."
- "Data: PostgreSQL for structured data, Redis for caching, S3 for everything else."

**Rules:**
- Pick 8-10 most impressive skills, not everything
- Recent skills matter more than old ones
- Context beats lists ("built 3 production APIs with FastAPI" > "FastAPI")
- Show breadth across categories (don't list 8 JavaScript frameworks)

### currently_learning
1-2 lines about what you're actively learning or exploring. Shows growth mindset.

- "Currently diving into Rust for a side project. The borrow checker is humbling."
- "Exploring event-driven architecture patterns. Coming from a REST-everything background."
- "Learning more about observability -- turns out logging isn't enough when things get distributed."

### cta_question
1 line inviting engagement about skills and learning.

- "What skill has been most valuable in your career that you didn't expect?"
- "What's on your 'to learn' list for this year?"
- "Fellow devs -- what technology surprised you the most when you finally tried it?"

### hashtags
3-5 relevant hashtags.

- Always include `#SoftwareEngineering`
- Add your primary stack tags: `#TypeScript`, `#Python`, etc.
- Add `#TechCareer` or `#DeveloperLife`
- Add `#BuildInPublic` or `#ContinuousLearning` if appropriate

## Constraints

- **Under 1300 characters total**
- **Focus on breadth with context** -- not a resume, not a list
- **First person** -- "I've built", "I'm learning", "My go-to is"
- **Professional but authentic** -- acknowledge strengths AND areas of growth
- **Don't list everything** -- 8-10 curated skills beat 25 bullet points
- **Frame with experience** -- "3 years of TypeScript across 5 projects" not just "TypeScript"
- **Show what you're learning** -- growth mindset is attractive
- **1-3 emojis max** -- use sparingly
- **Use line breaks liberally**

## Examples

### Example 1: Full-Stack Developer

#### Skills Evidence (not shown in post)
```
Repos analyzed: 4
Strong: TypeScript (4 repos), React (3 repos), Node.js (3 repos), PostgreSQL (3 repos)
Solid: Python (2 repos), Docker (2 repos), Redis (2 repos)
Emerging: Rust (1 repo), GraphQL (1 repo)
Resume additions: AWS (claimed), Terraform (claimed)
```

#### Post
```
Someone asked me "what's your stack?" last week. I decided to actually audit my repos instead of guessing. Here's what I found.

My core: TypeScript everywhere. React on the frontend, Node.js on the backend. It's been my daily driver for 4 years across dozens of projects.

Data layer: PostgreSQL is my default. I've learned to respect it -- proper indexing and query planning save more time than any framework choice. Redis for caching when I need speed.

Infra: Docker for everything. I don't deploy anything that isn't containerized anymore.

The surprise: I found more Python in my repos than I expected. Turns out I reach for it every time I need a quick script or data pipeline.

Currently exploring: Rust. Building a CLI tool as a learning project. The compiler is the strictest code reviewer I've ever had.

What's the one technology in your toolkit that you didn't plan to learn but can't imagine working without?

#TypeScript #React #Python #SoftwareEngineering #ContinuousLearning
```

**880 characters**

I spent this weekend mapping my actual skills against what I think my skills are. The gap was interesting.

Honest gap: frontend. I can modify a React component but building a UI from scratch isn't my strength.

Currently learning: Go. The simplicity is refreshing after years of Python's flexibility.

What's a skill you were honest about NOT being strong at, and how did that honesty help you?

#Python #DataEngineering #Backend #SoftwareEngineering #TechCareer
